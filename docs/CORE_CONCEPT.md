# LILIA Core Concept

## 一文定義

LILIAは、あなたとの会話・選択・物語を記憶し、
関係性と人格の出方が少しずつ変化していくAI恋愛シミュレーションです。

## LILIAとは

LILIAは、単なるAIチャットでも、固定シナリオの恋愛ゲームでもありません。

それぞれのLILIAは、固有の人格、価値観、弱さ、距離感、記憶の持ち方を持っています。

ユーザーとの会話、選択、日常、物語上の出来事を通じて、
LILIAは相手を知り、反応を変え、関係性の中で少しずつ変化していきます。

LILIAは、ユーザーに都合よく最適化される存在ではありません。
最初から完成された攻略対象でもありません。

LILIAは、記憶と関係の中で変化していくAI上の人格です。

## 価値提供

LILIAが提供する価値は、
ユーザーの言葉と選択が関係に残り、
長期記憶によって継続感が生まれ、
ストーリーイベントを通じて人格の出方と距離感が変化していく、
関係育成型のAI恋愛体験です。

ユーザーは、毎回リセットされる会話ではなく、
過去の出来事が次の会話に影響する関係を体験します。

一度きりの反応ではなく、
積み重なった記憶から生まれる態度、迷い、照れ、信頼、衝突、変化を体験します。

## ユーザーが得るもの

- 固有の人格を持つLILIAとの継続する関係
- 自分の言葉や選択が次回以降に残る感覚
- 日常の中に戻れる情緒的な居場所
- 物語を通じて関係が変化していく没入感
- LILIAとの思い出や関係の履歴
- 自分がどんな関係性に惹かれるのかを知る体験
- 育ったLILIAをキャラクターファイルとして持ち運べる可能性

## 解消するもの

LILIAは、既存のAIチャットや恋愛ゲームにある次の物足りなさを解消することを目指します。

- 会話が毎回リセットされる浅さ
- キャラクターの態度や人格が一貫しない違和感
- 固定シナリオ恋愛ゲームの選択肢の狭さ
- 既製キャラクターだけでは満たされない関係性への欲求
- 妄想や関係性を形にする手間
- 日常の中で感情を預けられる相手がいない感覚

## 事業としての軸

LILIAは、単なるAIチャットではありません。

LILIAは、ユーザーとの記憶を持ち、
人格の核を保ちながら、
関係の中で変化していくAI恋愛体験です。

固定された恋愛ADVではなく、
記憶・選択・物語によって関係が育っていく体験を提供します。

## ストーリーの役割

LILIAにおけるストーリーは、主役ではありません。

ストーリーは、LILIAの人格、距離感、信頼、迷い、嫉妬、甘え、警戒、開示を変化させるための出来事です。

事件や日常イベントは、単に解決されるために存在するのではありません。
LILIAが何を感じ、何を覚え、次にユーザーへどう向き合うかを変えるために存在します。

## 記憶の役割

記憶は、設定を増やすためのものではありません。

記憶は、関係の継続感を支えるためにあります。

LILIAは、重要な会話、選択、衝突、約束、沈黙、すれ違い、距離が変わった瞬間を記憶します。

それらの記憶は、次の会話での第一声、態度、距離感、言い淀み、照れ、信頼、警戒として現れます。

## LILIAの人格

各LILIAには固有の人格があります。

LILIAは、ユーザーの好みにただ迎合する存在ではありません。

LILIAには、好きなもの、嫌いなもの、怖いもの、守りたいもの、譲れないもの、言えない本音があります。

関係が深まることで変化する部分もあれば、変わってはいけない核もあります。

LILIAの魅力は、都合よく従うことではなく、
固有の人格を持った存在と関係を築いていくことにあります。

## 設計原則

- LILIAを所有物や攻略対象として扱わない
- 各LILIAには固有の人格がある
- ユーザーに迎合しすぎず、関係の中で変化する
- ストーリーは、関係と人格を変化させるための装置として扱う
- 長期記憶は、関係の継続感を支えるために使う
- 会話の温度、距離感、言い残し、次に会った時の反応を重視する
- 最初は1人のLILIAとの関係が面白いことを最優先する
- 育ったLILIAを持ち運べるキャラクターファイルとして扱えるようにする
- 事件・対策・構造の説明は、LILIAの声、仕草、温度を通して返す。システム解説として返さない。

## 短いコピー

あなたとの記憶で、関係が育つ。
