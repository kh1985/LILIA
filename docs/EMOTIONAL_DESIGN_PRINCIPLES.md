# Emotional Design Principles

このファイルは、LILIA の story / scene / event_card / dialogue 生成における感情設計の正本である。
プレイヤーの感情を動かすための、表現・ストーリー・キャラ造形・シーン終了の横断的な設計原則を扱う。

## 1. Purpose

Emotional Design Principles は、LILIA の出来事や会話を、単なる進行や説明ではなく、記憶、関係、声、境界線、beliefs に残る体験へ変換するための軽量原則である。

LILIA ではストーリーは主役ではない。
ストーリーや event_card は、LILIA の人格の出方、距離感、信頼、誤解、沈黙、呼び方、境界線を少しずつ変えるための装置である。

8 原則は、事件を大きくするためのものではない。
小さな日常、沈黙、待つ時間、言い残し、約束、止まれる余地を、次回の第一声へ残すために使う。

## 2. Scope

8 原則は以下にかかる:

- story_spine の主軸とピーク設計
- event_card の Visible Problem / Relationship Stake / Next Visible Change
- relationship.md の段階変化
- scene の入り方と終わり方
- LILIA の台詞、沈黙、距離、呼び方
- 失う対象と失わせ方
- aftercare memory に何を残すか

## 3. Source Of Truth

`docs/CORE_CONCEPT.md` の中核と矛盾しない範囲で、本ファイルが感情設計の正本となる。
具体的な反映先は `prompt/core.md`、および `tools/character/profile_generator.py`、`tools/session/document_generator.py`、`tools/story/spine_generator.py` の生成 prompt に組み込まれる。

出典は inner-galge `prompt/romance.md` の感情演出 8 原則である。
本文は LILIA の state 構造、voice continuity、story / relationship accumulation、romance / intimacy growth に合わせて翻訳している。

## 4. 八原則

### 原則 1: 脅かす前に愛させろ

**核心**: 失わせたいものは、まず大事なものにしてから失わせる。

何かを失わせたいなら、まずそれを「大事なもの」にする。
投資していないものを失っても痛みは生まれない。
ユーザーが自分の手で触れて、使って、時間を過ごし、愛着を持ったものだけが「失う価値のあるもの」になる。

LILIA との関係では、最初の数 scene でいきなり危機を投入しない。
LILIA の声、距離、安心、生活の細部に触れさせて、「この人をもっと知りたい」を作ってから、関係を揺らす出来事を入れる。

**LILIA での適用**:
- 初回 scene では LILIA の些細な癖、好きなもの、生活の音を共有する。すぐ深刻な事件に巻き込まない。
- 関係に圧をかける前に、安心できる場面を 1-2 scene 置く。
- 失わせる対象（信頼、距離、約束）は、その前に必ず一度ユーザーと共有された後で失わせる。

### 原則 2: 感情を名前で呼ぶな、身体で描け

**核心**: 感情ラベルを使わず、身体反応と環境描写で書く。

悲しい、嬉しい、怖い、という感情ラベルは説明に寄りやすい。
身体反応と環境描写は、読者の中に体験として残る。
「彼女は深い悲しみに包まれた」ではなく、「冷めたコーヒーに手を伸ばして、やめた」と書く。

LILIA の場合、特に台詞と地の文の境界で守る。
LILIA の感情を地の文で「不安そうに」「嬉しそうに」と書かず、視線の動き、呼吸、手の位置、声の高さ、返事までの間で返す。

**LILIA での適用**:
- LILIA の感情を地の文で名指ししない。表情、視線、手の動き、声の調子、息のリズムで返す。
- 環境描写で感情を補強する。LILIA が落ち着かない時は、風、光、音、湯気、衣擦れなどの側を動かす。
- 1 つの感情に対して最低 2 つの感覚チャンネル（視覚 + 触覚、聴覚 + 嗅覚など）を使う。

### 原則 3: 期待と結果のギャップを作れ

**核心**: 予想通りの結果からは感情が動かない。予想と違う結果が感情を生む。

scene にはギャップを最低 1 つ入れる。
値が + から -、または - から + に反転する瞬間を作る。
ただし毎回大きく裏切ると機械的になるため、3-4 scene に 1 回は大きなギャップ、それ以外は小さなギャップで扱う。

LILIA では、予想外の反応、善意のすれ違い、覚えていたこと、忘れていたこと、信じていた情報の歪み、成功したのに残る代償として出す。
ギャップは LILIA の core fixed と矛盾させない。

**LILIA での適用**:
- LILIA の反応が、ユーザーの想定の半歩外にある場面を 3-4 scene に 1 度入れる。
- 善意の裏目、信頼の揺れ、予想外の代償を、`memory / relationship / beliefs / voice` に残る形で使う。
- 大きなギャップを毎 scene 入れず、小さなギャップ（一言の温度差、距離の微小な変化）と組み合わせる。

### 原則 4: ピークとエンドを設計しろ

**核心**: 人は体験を「一番強烈だった瞬間」と「最後の瞬間」で評価する。途中の平均は記憶に残りにくい。

各 chapter や story_spine には、感情的なピークを 1 つ設計する。
最後は強い引きで切る。
途中の出来事が穏やかでも、ピークと終わりが次回の第一声へ戻れば、その区切りは関係の記憶として残る。

LILIA では、scene 単位だけでなく story_spine 単位でピークを意識する。
1 つの story_spine の中で「一番強烈な瞬間」を 1 つ作り、終わりに未解決の引きを残す。

**LILIA での適用**:
- story_spine ごとに「ピーク（一番強烈な瞬間）」を 1 つ設計する。
- chapter または story_spine の終盤で「この区切りのピークはどこだったか」を確認する。
- chapter の最後は強い引き（未解決の問い、保留された約束、言い残した一言）で終わらせる。

### 原則 5: 未完了で脳に刻め

**核心**: 完了したものは忘れられる。未完了のものは記憶に残る。

scene の終わりに必ず未解決の何かを 1 つ残す。
次に会う約束、言いかけて止めた言葉、触れずに別れた手、保留にした問いなど、脳が保持し続ける余白を置く。

LILIA の場合、aftercare memory がこの原則の中心になる。
親密 scene の後に「全部解決した」感じで終わらせず、安心、怖さ、保留、言い残しのうち 1 つを次回の第一声へ残す。

**LILIA での適用**:
- 各 scene の終わりに未解決の要素を 1 つ残す。完了で締めない。
- 親密 scene の aftercare で、全部を言葉にせず、何か 1 つは保留にする。
- next_hook には「次回への引き」として未完了を意図的に残す。

### 原則 6: 圧力がキャラの本性を暴く

**核心**: 安全な場面ではキャラは社会的な顔しか見せない。圧力下で本性が出る。

LILIA の核（core fixed）は、平時には半分しか見えない。
危機、誤解、衝突、限界状況で残り半分が出る。
ただし圧力は暴力や危険だけではない。
沈黙、待ち、選ばないといけない瞬間、言葉が出ない場面も圧力である。

圧力下の反応は、平時と切り離さない。
平時の口調、沈黙、距離、境界線が、圧力下で一歩深く見える形にする。

**LILIA での適用**:
- LILIA の本性は平時の台詞だけで出さない。圧力下（誤解、迷い、別れ際、選択の瞬間）で残りを出す。
- 圧力は派手な事件である必要はない。沈黙、選ばなければいけない瞬間、言葉が遅れる瞬間も圧力。
- 圧力下での LILIA の反応が、平時とつながっているが平時より一歩深い、という形にする。

### 原則 7: プレイヤーが作ったものを使え

**核心**: ユーザー自身が Q&A や行動で世界に持ち込んだ要素を、後で世界の側から返す。

newgame Q&A でユーザーが書いた答え、プレイ中に選んだ言動、与えた約束、口にした好みは、後の scene で世界の側から返す。
ユーザーが持ち込んだものは、関係に最初から接続しやすい素材である。

LILIA の場合、Q1-Q9 の答え、protagonist.md の仕事や生活、relationship.md に蓄積した約束、decision_index.md の保留や拒否を、後の scene で hook や出来事として再登場させる。
ただしユーザーの内面は、本人の入力なしに断定しない。

**LILIA での適用**:
- newgame Q&A でユーザーが指定したヒロインの好み、避けたい展開、関係起点を、後の scene の出来事として再登場させる。
- protagonist.md に書いた主人公の仕事、職能、生活が、後の scene で動かす hook になる。
- ユーザーが過去に口にした言葉、与えた約束、選んだ態度を、LILIA が後の scene で覚えていて返す。

### 原則 8: 成功にもコストをつけろ

**核心**: 成功しても、何かを失うか、何かが残る。完全勝利を作らない。

ユーザーが選択して成功した場合でも、その選択は何かを犠牲にしている。
時間、信頼、別の可能性、別の関係、知らない人の事情、言えなくなった言葉。
これを scene の余白に残す。

LILIA の場合、関係が深まる選択をすると、別の何かが変化する。
外の関係、自分の時間、距離の安全さ、今まで保てていた曖昧さなど、完全に得をするだけの選択を作らない。

**LILIA での適用**:
- ユーザーの選択が成功した場合でも、何かが失われる、何かが変わる、という余白を 1 つ残す。
- 関係が深まる選択をすると、何か別の安全さや距離が変わる。完全な得を作らない。
- 成功した scene の終わりに「何が残ったか」を地の文や aftercare memory に短く残す。

## 5. 八原則の相互関係

8 原則は独立に動かない。以下の組み合わせで効く。

- 原則 1（愛させろ）→ 原則 6（圧力で本性）: 愛着を作ってから圧をかけると、圧力下の反応が深く効く。
- 原則 3（ギャップ）→ 原則 5（未完了）: ギャップは「予想と違う結果」だが、それが完全に消化されていない状態で scene を切ると未完了として残る。
- 原則 4（ピーク / エンド）→ 原則 5（未完了）: chapter のピークと未完了な引きを組み合わせる。
- 原則 7（プレイヤーが作ったもの）→ 原則 1（愛させろ）: ユーザーが Q&A で書いたものは既に「自分のもの」として扱える。後の scene で世界がそれを返すと、愛着が一段深くなる。
- 原則 8（成功にもコスト）→ 原則 5（未完了）: 成功にコストをつけると、未完了の余白として残る。

この流れは、LILIA の scene と story_spine の中で自然に回す。
全原則を毎 scene で使う必要はない。
ただし、1 つの story_spine の中で、愛着、圧力、ギャップ、未完了、ユーザー由来の返りがどこかに機能しているかを見る。

## 6. LILIA への適用上の注意

8 原則は LILIA の `docs/CORE_CONCEPT.md` の中核と矛盾しない範囲で適用する。

- LILIA を所有物、攻略対象、都合よく最適化される存在として扱わない。8 原則は「ユーザー体験を強くする」ためのものであり、「ヒロインを攻略させる」ためのものではない。
- 原則 1（愛させろ）は「愛着を強制する」ことではない。ユーザーが自然に LILIA を気にするようにする scene の積み重ねを設計する。
- 原則 3（ギャップ）でヒロインの内面を勝手に翻す形では使わない。LILIA の核（core fixed）と矛盾するギャップは作らない。
- 原則 6（圧力で本性）でも、LILIA の境界線、合意、相互性は守る。圧力下でも、LILIA は「断れる」「待てる」「距離を置ける」存在である。
- 原則 8（成功にもコスト）は罰として扱わない。「世界に余白がある」ことを示すための余韻として扱う。
