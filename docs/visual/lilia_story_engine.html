Causal Loop Diagram

状態が次の物語を生む循環

1つのsceneの結果は保存され、次のsceneで前景化される入口を作る。

  1. 1. User Actionプレイヤーの発言、行動、沈黙
  2. 2. Active Hook今sceneで触れる入口
  3. 3. Dramatic Question今追う問い
  4. 4. Character Want / Fear / Misbeliefキャラが欲しいもの、恐れているもの、誤って信じているもの
  5. 5. Choice Tensionどちらを選んでも何かが変わる迷い
  6. 6. Causality行動が関係・情報・状況にどう作用するか
  7. 7. Irreversible Delta元に戻らない変化
  8. 8. State Updatememory / relationship / beliefs / voice への保存候補
  9. 9. Next Curiosity次に知りたくなる問い
  10. 10. Candidate Hook次scene候補
  11. 11. Next Active Hook次に前景化される入口
LILIAでは、出来事そのものではなく、出来事によって変わった状態が次の物語を生む。

Three Story Layers

筋立て・感情・意味

同じsceneでも、筋立てだけが動くと説明になる。感情と意味まで少し変わると関係の物語になる。

Plot Layer / 筋立て

担当

  • scene
  • event_card
  • story_deck
  • active hook

問い

  • 何が起きるか
  • 次に何をするか
  • 何が普通ではないか

Emotion Layer / 感情

担当

  • state
  • relationship
  • memory
  • beliefs
  • voice

問い

  • 誰が何を望むか
  • 何を恐れるか
  • どう受け取るか
  • 関係がどう揺れるか

Meaning Layer / 意味

担当

  • relationship_spine
  • core
  • theme
  • contradiction

問い

  • この関係は何を問うのか
  • 何が変わると物語として意味があるのか
  • どんな価値観がぶつかるのか
面白いscene = 筋立てが動く + 感情が揺れる + 意味が少し変わる

Story Function Ladder

物語を進める15機能

これは固定プロット順ではなく、sceneの役割を診断するための道具。

  2. 2. 前提どんな暮らし・ルールの上にあるか
  3. 3. 起点何が日常を揺らすか
  4. 4. 目標何を求めて動くか
  5. 5. 始動なぜ一歩踏み出すか
  6. 6. 試練何が難しいか
  7. 7. 糸口次へ進む足場
  8. 8. 前進何が少し進んだか
  9. 9. 転回問題の意味がどう変わったか
  10. 10. 孤立何を失い、自分で選ぶか
  11. 11. 奈落関係や心が最も沈む点
  12. 12. 打開積み上げたものが突破口になる
  13. 13. 選択中心問題にどう答えるか

15機能は順番どおりに埋める表ではない。今のsceneがどの役割を担うべきかを見る診断レンズ。

2. One Turn Pipeline

1ターン出力の流れ

状態ファイルを全部本文に出すのではない。 今この瞬間に必要な一手へ変換する。
1ユーザー入力を読む
2scene.md で現在地を確認
3event_card.md で今触れる入口を確認
4voice / state / relationship / memory / beliefs を参照
5Turn Focus を1つに絞る
6ヒロインの反応を作る
7状況・関係・情報のどれかを1つ動かす
8次に返せる入口を残す

4. Two Safety Gates

分かりやすさと面白さ

この2つを分けると、状態、hook、event_card、story_deckの役割が見えやすくなる。

Player Orientation Gate

分かりやすさのGate

迷子にさせない

真相は隠してよい。ただし、話の入口は隠さない。

  • これは何の話か分かるか
  • 主人公がなぜ関係者なのか分かるか
  • 今何が普通ではないのか分かるか
  • 何に触れば進むか分かるか
  • 判断を迫る前に判断材料があるか

NG意味深な反応だけが続き、何が問題か分からない

OK真相は不明だが、次に確認すべき対象は分かる

Story Causality Gate

面白さのGate

状態が物語を動かす

分かるだけでは弱い。選択によって、何かが不可逆に変わる必要がある。

  • 誰が何を欲しがっているか
  • 何が邪魔しているか
  • 選択に代償があるか
  • ユーザー行動が何に作用するか
  • ヒロインの受け取り方が変わるか
  • 次に見たい問いが残るか

NG状況説明は分かるが、何も変わらない

OK選択によって、信頼・警戒・情報・約束・境界線のどれかが変わる

6. Bad vs Good Flow

悪い流れ / 良い流れ

特定セッションに依存しない、構造だけの比較。

Bad Flow

  1. 怪しい出来事
  2. ヒロインが意味深に反応
  3. 謎が増える
  4. 選択を迫られる
  5. プレイヤーが何を判断すればいいか分からない

Good Flow

  1. 怪しい出来事
  2. 何が普通ではないか分かる
  3. 主人公がなぜ関係者か分かる
  4. 選べる行動が見える
  5. 選択に代償がある
  6. 関係・情報・状況のどれかが不可逆に変わる
  7. 次に見たい問いが残る
