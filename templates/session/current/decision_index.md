# Decision Index

セッション中にユーザーとLILIAの間で「決まったこと」を保存する索引。
何が起きたか（memory.md historical_fixed）とは別に管理する。
これにより、解決済みの約束・拒否・保留が、再開時に未完了扱いされることを防ぐ。

Save Mode でのみ更新する。Play Mode 中は更新しない。
古い決定は上書きしない。新しい状態として追記する（撤回・変更は明記）。

## 約束（Promises）

ユーザーまたはLILIAが「次にこうする」「いつかこうする」と言ったこと。

- 約束1:
  - 内容:
  - 誰が誰に:
  - 期限 / 条件:
  - 状態: active / fulfilled / broken / withdrawn
  - 関係への影響:

## 拒否（Refusals）

ユーザーまたはLILIAが「これはしない」「これは触れない」と表明したこと。

- 拒否1:
  - 内容:
  - 誰が:
  - 理由 (optional):
  - 状態: active / withdrawn

## 保留（Deferrals）

「今は決めない」「後で話す」と保留にしたこと。

- 保留1:
  - 内容:
  - 誰が:
  - 戻る条件:
  - 状態: pending / resolved / abandoned

## 解決済み（Resolved）

完了・解決した事項。再開時に未完了扱いしない。

- 解決1:
  - 内容:
  - 解決の経緯:
  - 関係に残ったもの:

## 注

- 撤回された約束・拒否は削除せず、状態を `withdrawn` にする。
- ユーザーの内面を断定しない。「ユーザーは〜と決めた」と書く時、本人の入力に基づくものだけ。
- LILIAの拒否・保留はLILIA側仮説（beliefs.md）ではなく、実際に表明された言葉だけを書く。
- 軽い意向や匂わせは書かない。明示された決定だけ書く。
