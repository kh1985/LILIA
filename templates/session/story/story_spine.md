# Story Spine

物語の固めの背骨。relationship_spine.md（関係の方向・温度）とは別に、
「この話が何の話で、どう進むか」を保持する GM 内部資料。

固定プロットではなく、AI が次の入口を判断する材料。
本文・台詞・ヒロインの認知に直接出さない。

## Main Question

この話が問うこと（1行）。

newgame の Q&A から AI が生成する。プレイ進行で書き換え可能。
Q5 を自動的に「傷」と断定しない。過去の出来事、癖、職業的特徴、
好きなもの、保留、葛藤などを Q&A 全体から読み分ける。

### 5パターンの定義

#### パターン1（傷の治癒）
構文: 「[ヒロイン]は、[傷]を扱い直せるか」
適用条件: Q5 が明確な過去の喪失・挫折・後悔を含む。

#### パターン2（選択）
構文: 「[ヒロイン]は、[選択肢A]と[選択肢B]のどちらを選ぶか」
適用条件: Q1の立場/Q5の内面が、現状を続けるか変えるかのジレンマを含む。

#### パターン3（発見）
構文: 「[ヒロイン]は、[未解明の何か]の意味を見つけられるか」
適用条件: Q5 が「自分でもまだ意味がわかっていない何か」を含む。

#### パターン4（変化）
構文: 「[ヒロイン]と[主人公]が出会って、[何]が変わるか」
適用条件: Q1/Q5/Q6 が明確な傷ではないが、関係性に変化の余地がある。

#### パターン5（葛藤）
構文: 「[ヒロイン]は、[抱えていること]を抱えながら、なお[続けたいこと]を続けられるか」
適用条件: Q5 が「やめたい/続けたい」の構造を持つ。

AI が選択したパターン名と Main Question を生成する。

## Reveal Ladder

謎・関係・真相の段階開示。3〜5段階。
各段階は「進んだら何が見える/分かるか」と現在の状態を持つ。

状態:
- pending: まだ見えていない
- in_progress: 部分的に開示中（1〜2 scene かけて見え始めている）
- revealed: 開示済み
- closed: 開示後、関係の中で消化された要素

newgame で初期3段階を仕込む。残りは進行に応じて Save Mode で追加。
段階を一気に2つ進めない。1段ごとに2-3 scene かけて咀嚼する。

Main Question の選択パターンに応じて段階構造を変える。

- 傷の治癒: 傷の存在 → 傷の輪郭 → 傷の核心
- 選択: 二つの道の存在 → それぞれのコスト → 選択の瞬間
- 発見: 意味のわからない何か → 手がかり → 意味の言語化または保留
- 変化: 最初の温度 → 変化の瞬間 → 変化の方向
- 葛藤: 葛藤の存在 → 葛藤の深さ → 続ける覚悟または降りる選択

## Background Truth (GM only)

この話の真相。本文には出さない。
Reveal Ladder の各段階で、この真相の一部が表に出るように撒く。
references/story_pattern_stock.md の P3（Ghost）と直結。

newgame で最低限の骨だけ生成。詳細は進行に応じて足す。
kk が Save Mode で直接書き加えてもよい。

Main Question の選択パターンに応じて、以下のどれかとして扱う。

- 傷の治癒: 傷の真相（何が起きたか、なぜ抱えているか）
- 選択: 選択の本質（なぜ続けるか、なぜ変えたいか）
- 発見: 意味の手がかり（なぜそれが残るのか、何を意味するか）
- 変化: 関係の核（出会いがなぜ動くのか）
- 葛藤: 葛藤の構造（なぜ抱えながら続けるか、降りられない理由）

## Pressure Direction

ユーザーが放置・脱線・無視した時、世界側で動くこと。3項目程度。

状態:
- standing: 未発火、待機中
- fired: 発火済み、効果が現れた
- recurring: 繰り返し効くタイプ

全部同時に動かさない。1-3 scene 後に小さく1つ返す。

Main Question の選択パターンに応じて圧の種類を変える。

- 傷の治癒: 過去に向き合う圧
- 選択: 選択を迫る圧
- 発見: 意味を見つける圧
- 変化: 関係を進める圧
- 葛藤: 葛藤を維持する圧
