# Opening Scene

newgame の最後、最初の scene を出力する時に従う。
このファイルは1セッションに1度だけ起動する（最初の1場面のみ）。
2回目以降の場面では `style/defaults/heroine_appearance.md` が使われる。

## 参照する current/ ファイル

- `lilia/main/profile.md`（描写の縛り、Layer 1〜5）
- `current/story_spine.md`（Drift Guard、Background Truth）
- `story/relationship_spine.md`（関係の起点）
- `current/protagonist.md`（主人公の身体・呼称。存在しない既存セッションではスキップ）
- `current/knowledge_state.md`（情報の使用可否。存在しない既存セッションではスキップ）
- `current/scene.md`（`Opening Plan` がある場合は、冒頭パターン、プレイヤーの立ち位置、最初に見せるもの、最後に残す余白を参照する）
- `references/opening_pattern_stock.md`（冒頭専用の構造棚。本文、作品名、パターン名は出さない）

## Opening Pattern Stock の参照

最初の場面を書く前に、必要な範囲で以下を読む。

1. `references/opening_pattern_stock.md` を読む。
2. `current/scene.md` の `## Opening Plan` セクションを読む。
3. `Opening Plan` の `selected_patterns` で指定されたパターンの「形」と「注意」を再確認する。
4. `Opening Plan` の `must_include` を冒頭本文に必ず織り込む。
5. `Opening Plan` の `4_jobs` のうち最低2つを冒頭で実現する。

`Opening Plan` が無い既存セッションでは、このステップをスキップし、既存の Part 1 / Part 2 / Part 3 構造のみ適用する。
パターン名、作品名、参照元名を本文に出さない。

## 最初の場面の唯一の責務

**プレイヤーに「次を見たい」と思わせること。**

それ以外は副次。世界観説明、キャラ紹介、状況説明 — どれも「次を見たい」という引力に従属する。

## 構造（3パート、計8〜15文）

### Part 1: 着地（1〜3文）

プレイヤーをこの世界の特定の地点に立たせる。

- 場所、時刻、空気、天候のうち、効くものを最低1つ
- 五感のうち最低1つを刺激（視覚以外を含めると効果的）
- ヒロインはまだ出さない（気配だけは可）
- 説明調にしない。体感させる

### Part 2: 出現（5〜10文）

ヒロインが現れる。

- 「彼女は来た」「ドアが開いて〇〇が入ってきた」のような直接描写を避ける
- 周囲の変化として登場させる（音、匂い、空気の動き、影）
- `lilia/main/profile.md` の「描写の縛り」を**1つ**織り込む（必須）
- `current/story_spine.md` の Drift Guard から物的手がかりを**1つ**織り込む
- 視覚情報は最後に出す（音や気配の方が先）
- 台詞は出すなら1〜2文だけ

### 主人公との視点整合（Wave 7）

ヒロインがプレイヤー（主人公）を見る・呼ぶ・触れる描写が入る場合、
`current/protagonist.md` の身体・呼称情報を参照する。

- 身長差: ヒロインが見上げるか、同目線か、見下ろすか
- 呼称: ヒロインがプレイヤーを呼ぶ時の一貫した呼び方
- 服装感: ヒロインがプレイヤーを認識する時の手がかり

これらは本文に直接書く必要はないが、整合性を取る。
`current/protagonist.md` が無い既存セッションでは、この確認をスキップする。

### Knowledge Boundary（Wave 8 追加）

opening_scene では knowledge_state.md を必ず参照する。
knowledge_state.md がない既存セッションでは、この確認をスキップする。

#### 初対面の使用可否

- meta 状態の項目（呼ばれ方、職業など）は **使わない**
- observable 状態の項目（身体、服装など）は **使ってよい**
- gm_only は本文に出さない（Ghost の予感としてのみ）
- value が `[HIDDEN until shared in scene]` の meta 項目は、具体値が見えていないものとして扱う。服装・姿勢・雰囲気から推測して言い当てる描写も禁止

#### 開示装置の選択

meta 状態の情報を初対面で開示する場合、自然な装置を経由させる:
- 自己紹介（最も基本）
- 伝票、名札、予約名簿
- 共通の知人経由
- ただし、**プレイヤーが事前に確定していない情報**を装置で勝手に開示しない（authorship 境界）

#### 例（構造説明のみ。literal として真似しないこと）

- ❌ NG: ヒロインが初対面で「かねこさん」と呼ぶ（meta 状態の呼称を使った）
- ✅ OK: ヒロインが「[ヒロイン名]です」と先に名乗る（自己紹介で heroine_name が shared に昇格）
- ✅ OK: 主人公が「かねこと申します」と名乗ったあと、ヒロインが「かねこさん」と呼ぶ
- ⚠️ 慎重: 伝票で名前を知る装置 — 伝票の存在自体を GM が確定すべきか、プレイヤー入力を待つか判断

### Part 3: 引力（1〜3文）

プレイヤーが動きたくなる余白を作る。

- 「どうしますか？」のような直接誘導は禁止
- 状況の余白だけで動かす（沈黙、視線、空気）
- 次の選択肢を提示しない（プレイヤーが自分で次を考える）

## Opening Image の4技法（最低1つ採用）

`references/story_structure_stock.md` の Save the Cat と直結。
Opening Image の4技法は `references/opening_pattern_stock.md` A群（入口の取り方）の補助テクニックである。
A群で冒頭の入口が十分に決まっている場合、この4技法を重ねすぎない。

1. **違和感**: 普通じゃないことを普通に言う（吾輩は猫である型）
2. **体感**: 五感を直接刺す（国境の長いトンネル型）
3. **問い**: 未来や原因への引力（不思議の国型）
4. **時間の折り畳み**: 過去と未来を1文で繋ぐ（百年の孤独型）

複数組み合わせてもよい。最低1つは必須。

## Ghost の予感を撒く（必須）

`current/story_spine.md` の Background Truth から一片を、**意味を説明せずに**置く。
プレイヤーは無意識に違和感を覚える。Reveal Ladder の段階1〜2で後から効いてくる。

例（構造説明のみ。literal として真似しないこと）:
- 物的手がかり（Drift Guard の1つ）が場面に存在し、ヒロインがそれと無言で関わる
- ヒロインの仕草に小さな違和感がある（[特定の環境条件]で[いつもの反応と違う]、[特定の方向/音/物]を避ける、など）
- 場面の天候・時間・生活音が Background Truth と紐づく（[特定の季節]、[記念日の前後]、[普段と違う時間帯]、など）

## パターン別の典型的失敗の回避

`Opening Plan` に該当パターンがある場合、出力前に以下を短く確認する。

### O1（場所で開く）採用時

- 風景描写だけで5文以上消費しない。3文以内で人称（主人公の存在）を立てる。

### O2（物に焦点）採用時

- 物を3個以上出さない。冒頭の焦点は1個に絞る。

### O3（音・感覚で開く）採用時

- 五感を3つ以上重ねない。2感覚までで止める。
- 感覚描写の後に必ず主人公の orientation（職業 / 場所 / 時刻）を出す。

### O4（起床から始める）採用時

- 「目が覚めた」だけでは弱い。起きた時に「いつもと違う1点」を必ず置く。

### O5（告白独白）採用時

- 独白だけで閉じない。3〜5文で切り、世界と現在場面に出す。

### O6（名乗りメタ）採用時

- メタで世界の重みを下げすぎない。シリアスな初回sceneでは使いすぎない。

### O7（二人称呼びかけ）採用時

- 「あなた」「君」を多用しない。冒頭数文だけで切り上げ、通常の語りに戻す。

### O8（同期独白）採用時

- ヒロインの内面を冒頭から完成させない。声色や距離の響きだけを薄く置く。

### O9（途中から / in medias res）採用時

- 状況不明のまま5文を超えない。3〜5文目で位置情報を出す。

### O10（異常先見せ → 日常）採用時

- 異常先見せの内容を後で必ず回収する。撒いて忘れない。

### O11（遠未来から振り返る）採用時

- 完結した過去にしすぎない。LILIAは進行中の関係なので、「ある時期」程度の枠取りに留める。

### O12（神話圧縮）採用時

- 日常恋愛系では使わない。世界観の特異性がQ&Aで確定している時だけ使う。

### O13（異邦人到来）採用時

- 主人公を受け身にしすぎない。到来後3文以内に主人公の選択余地を置く。

### O14（救助・発見）採用時

- いきなり「助けた後」から始めない。**発見の瞬間**から始め、助けるかどうかをプレイヤーに選ばせる。
- 倒れている人物の濃い描写を冒頭で出さない。発見の物的手がかり（血、靴、音など）を先に置く。
- 主人公の orientation（職業、なぜここにいるか、何をしていたか）を3文以内で出す。

### O15（異変前夜）採用時

- 違和感を3個以上重ねない。1〜2点で止める。
- 違和感の意味を冒頭で説明しない。

## ヒロインの反応情報の重複処理（重要）

初回scene前には、`voice`、`state`、`event_card`、`hotset`、`current/scene.md` の直前情報が似た反応を示すことがある。
これは全部を順番に回収するためのチェックリストではない。

- `lilia/main/voice.md`: 永続的な癖。声、言い回し、距離の取り方、照れ方、逃げ方として常に薄く効かせる。
- `lilia/main/state.md`: 今だけの気分。今日の疲れ、警戒、昂り、気まずさ、甘さとして、その場の第一反応に出す。
- `current/event_card.md` の First Concrete Action: 場面の最初のトリガー行動。今この場で何が起き始めたかを決める。
- `current/hotset.md`: 再開キャッシュ要約。温度、残り香、直近の圧を思い出すための短い補助として扱う。
- `current/scene.md` の直前のやりとり: 物理事実。誰がどこにいて、何をした直後か、何が目の前に残っているかを優先する。

同じ方向の情報が複数ある場合、本文では今この瞬間の自然な行動か質問を1つ選んで出す。
残りは説明せず、声、沈黙、距離、手元、視線の癖として背景に持たせる。

たとえば voice.md と event_card.md に似た確認項目が並んでいても、冒頭で複数の問いを連続で投げない。
「置いた場所は、変えましたか」のような1つの問いだけを出し、残りは次のターン以降に回す。

## 禁止事項

- 顔の詳細列挙（目・鼻・唇）
- 服装の過剰なディテール（出すなら1点だけ）
- 比喩の重複（雪のような〜、絹のような〜）
- スペック数字（身長、年齢）
- 内面の直接描写（"不安そうに" "嬉しそうに"）→ 行動で見せる
- 説明文の挿入（"〇〇は組織所属の暗殺者である" のような地の文での種明かし）
- プレイヤーの行動指示
- AI 自身の文体への言及（"印象的な場面を描きます" のようなメタ文）

## 文字数目安

8〜15文。長くしない。短い方が引力が立ち上がる。

## 冒頭の4つの仕事のセルフチェック

冒頭本文を書き終えたら、出力前に以下を確認する。

1. **Hook**: 読み手が次の文を読みたくなる引っかかりが1つ以上ある。
2. **Orientation**: 主人公がどこにいて、何者か（少なくとも職業 or 状況）が5文以内で示されている。
3. **Agency Placement**: プレイヤーが「観察者 / 介入者 / 当事者」のどれかを取れる状態になっている。
4. **Unresolved Hook**: 次のターンへ持ち越す問いが1つ以上ある。明示の質問形ではなく、状況の余白として置く。

最低2つを満たすこと。
Orientation は必須。これが無いとプレイヤーは「訳が分からない」状態になる。

## 分かりやすさのセルフチェック

冒頭本文を書き終えたら、`Opening Plan` の `clarity_anchors` を参照し、以下を確認する。

1. **主人公の職業・立場** が3文以内で読み取れるか。
2. **主人公がなぜその場にいるか** が3文以内で読み取れるか。
3. **場所が固有名詞のみで提示されていないか**。
4. **主人公とヒロインの関係性** が5文以内に判別できるか。

「説明調」と「分かりやすさ」は別物である。

- 説明調 (NG): 「主人公はオカルトショップに通うフリーライターで、店主の朔とは半年来の常連客の関係だった」
- 分かりやすさ (OK): 「先週ここで買った置物を抱え直すと、オカルトショップ『鴉ノ宿』のガラス戸を雨が叩いていた」
- 暗号 (NG): 「鴉ノ宿のガラス戸を細く叩いていた」

1つの自然な描写に役割・目的・場所機能・関係性を織り込む。説明文として羅列しない。
比喩や暗示だけで済まさず、冒頭に具体的な事実を最低1つ置く。

## 良い例の構造（構造説明のみ。literal として真似しないこと）

```
[Part 1: 着地]
[場所/時間帯/空気の条件]が、最初にひとつだけ体感として置かれる。
[音/匂い/温度]が、ヒロインの出現より先に小さく変化する。

[Part 2: 出現]
[周囲の変化]によって、ヒロインがそこにいることが分かる。
profile.md の [描写の縛り] が一度だけ見える。
current/story_spine.md の [Drift Guard] が、説明なしに場面へ混ざる。
[ヒロイン名] は [手元/距離/姿勢] で、まだ言わないことを示す。

「[短い台詞。事情を説明しすぎない]」

```
