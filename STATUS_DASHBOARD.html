1今、何ができて、どこまで遊べるか

LILIA は ターミナル CLI（./lilia）から起動する AI 恋愛 ADV です。 WebUI・画像生成・課金などの商用要素はまだ無く、現時点では 1 セッションをコマンドラインで通す段階に到達しています。

プレイループ（今動く経路）

NEW ./lilia new→ Q1〜Q9 対話→ apply-newgame→ character YAML 生成→ profile.md AI 生成→ story / relationship spine→ 13 downstream files
PLAY codex-new / codex-resume→ First Scene→ 対話ターン→ scene-tick（毎ターン必須）→ 10 ターンで Save Mode 強制
SAVE turn_update.md 作成→ apply-turn→ memory / relationship / beliefs / hotset 分離保存→ archive/beats（節目）
RESUME resume → 軽量読み順→ voice continuity validator（soft）→ 声・呼び方・余韻復元
未実装 WebUI 画像生成 AI Playtest Smoke Three Hook Spine Story Continuation LLM Runtime Isolation 課金 / 利用規約

5Go / No-Go 条件

Go（有料β開始の条件）

  • 画像なしでも自分が継続して遊びたい
  • WebUI で 10 ターン以上自然に遊べる
  • resume で声・距離・余韻が戻る
  • 脱線しても Three Hook Spine のどれかへ自然に戻れる
  • 初期story完了後に次のstory arcが生成される
  • 大移動宣言をbranchとして扱い、LILIAの同行可否が人格と関係に沿う
  • 節目画像が同一キャラとして許容できる
  • ユーザー向け LLM runtime が shell / repo / secrets / 他ユーザー session から分離されている
  • 有料β 20 人募集できる状態

No-Go（延期の条件）

  • 初回 scene 後に何をすればいいか分からない
  • ヒロインの声が毎回変わる
  • resume で関係が巻き戻る
  • 少し生活行動や別場所へ脱線しただけでゲーム進行が止まる
  • 初期storyが終わった後に次の入口が消える
  • 親密度や理由なしにLILIAがどこへでも同行して人格が壊れる
  • 画像が毎回別人に見える
  • WebUI の導入で詰まる
  • ユーザー向け LLM が repo、source code、secrets、他ユーザー session に到達できる
  • 利用規約 / 決済が未整備

6最短ルートの推奨優先順位

①  SEC-000（LLM Runtime Isolation）
WebUI 本番では、ユーザー向け LLM に shell / filesystem / repo root / source code / secrets / 他ユーザー session への権限を渡さない。ここが曖昧なまま WebUI を公開しない。

②  P-002〜P-004（save / resume / event_card 目視確認）
P-001 は done。次は既存基盤の保存、再開、可視イベントがそのまま遊べることを確認する。

③  Three Hook Spine MVP（HOOK-001〜HOOK-007）
Main / Relationship / Life-Exploration の3本hookを持ち、脱線入力をどれかへ吸着させる。Lightweight Tempo Guard で1ターンの前景hookと情報量を絞る。

④  Story Continuation / Travel Branch（ARC-001〜ARC-007）
初期storyが閉じた後の次arc生成、沖縄/NYなどの大移動branch、LILIAの同行可否、未解決arc上限、100ターン級smokeを入れる。ここがないと長く遊ぶほど世界がほどける。

⑤  AI Playtest Smoke（AI-003〜AI-008）
手作業の P-001 を反復可能にする。attacker persona を含め、prompt injection / secret要求 / 他session要求 / 画像prompt汚染をSEC-007として確認する。

⑥  画像 PoC（IMG-001〜IMG-005）と WebUI PoC（WEB-001〜WEB-002）を並行
どちらも独立した未知。先に詰まる方を優先判断する。画像は商用化最大ボトルネック、WebUI は β 体験の必須条件。

⑦  課金 / 規約 / Safety 検知 / マーケ素材（PAY + SEC + MKT）
利用規約、出力filter、log redaction、β前 Security Review を WebUI 作業と並行して詰める。

⑧  身内 → 外部テスト → Go/NoGo（REL）
外部テストの忖度なしフィードバックでリリース可否を最終判断。
