# Story Structure Stock

物語構造の参考カタログ。GM（AI）が「物語が今どこにいるか」「次に何を見せるか」を診断する時の足場。

## 利用ルール

- 構造を「正解」として使わない。**現在のセッションを診断する道具**として使う。
- 1セッションに1つの構造で十分。混ぜると当てはめのための当てはめになる。
- LILIA は「既に進行中の関係」だから、構造の途中から当てはめてよい。
- 構造はキャラの台詞や本文に出さない。GM の内部ノートとして使う。
- 構造はセッションを「止める」ものではない。「今どの位置にいるか」を見るもの。
- 例は構造説明のみ。literal として真似しないこと。

---

## 1. Story Circle（Dan Harmon）8段階 — 最も汎用的

LILIA に最も向く。キャラ中心、循環構造。1場面でも、1ヶ月の関係単位でも、半年単位でも、同じ8段階が回る。

| 段 | 名前 | 内容 |
|---|------|------|
| 1 | You | キャラが慣れた場所にいる |
| 2 | Need | 何かを欲する/必要とする |
| 3 | Go | 慣れた場所から踏み出す |
| 4 | Search | 適応しようとする/もがく |
| 5 | Find | 望んだものを得る |
| 6 | Take | 代償を払う |
| 7 | Return | 元の場所に戻る |
| 8 | Change | 戻った時、変わっている |

### 当てはめ例（構造説明のみ。literal として真似しないこと）

#### [ヒロインA] — 世話役を降りる関係
- You: いつも誰かの相談を受ける生活。
- Need: 自分が何も答えなくていい場所。
- Go: [ユーザー]の前で初めて返事を保留する。
- Search: 頼ることを試す。
- Find: 急かされない安心を一瞬得る。
- Take: 相談役でいられない怖さが出る。
- Return: いつもの生活に戻る。
- Change: 戻っても、次は自分から少し話す。

#### [ヒロインB] — 記録を信じられない関係
- You: 正確な記録を支えにしている。
- Need: 記録に残らない実感。
- Go: 曖昧な記憶を[ユーザー]に話す。
- Search: 確認するか、黙るかで揺れる。
- Find: 覚えていてもらえた安心を得る。
- Take: 自分の記録との矛盾を受ける。
- Return: 記録の場所へ戻る。
- Change: 記録だけを正本にしなくなる。

#### [ヒロインC] — 触れない境界の関係
- You: 一定距離を守る生活。
- Need: 近づいても止まれる確信。
- Go: 境界線を言葉にする。
- Search: 待つ、断る、近づくを試す。
- Find: 触れないまま近い時間を得る。
- Take: 近づきたい気持ちと怖さが衝突する。
- Return: いつもの距離に戻る。
- Change: 戻った距離の意味が変わる。

このサイクルは関係の「段」ごとに1回ずつ回る。今がどの段にいるかを見れば、次に何を見せるべきかが分かる。

### 使い時
- 関係全体の進行を診断したい時。
- 「今、進んでいる感じがしない」と感じた時、どの段で停滞しているかを見る。

## 2. Save the Cat 15ビート — パチッと山と谷を作る

Story Circle より細かい。映画的な構造。長期的な関係（10セッション以上）を見渡す時に有効。

LILIA 向け圧縮版（重要ビートだけ）:

| % | ビート | 内容 |
|---|--------|------|
| 1 | Opening Image | 関係前の世界 |
| 10 | Catalyst | 関係を起動する出来事 |
| 10-20 | Debate | 行くか戻るか迷う |
| 20-25 | Break into Two | 関係に踏み出す決断 |
| 22 | B Story | 別軸で核心に近づくキャラ/関係 |
| 50 | Midpoint | 関係の偽の勝利 or 偽の敗北 |
| 75 | All Is Lost | 関係の最低点 |
| 80 | Dark Night of the Soul | 諦めの時間 |
| 80-85 | Break into Three | 戻る決意 |
| 85-99 | Finale | 統合 |
| 99 | Final Image | Opening Image との対比 |

### 当てはめ例（構造説明のみ。literal として真似しないこと）

#### [ヒロインA] — 生活儀式の変化
- Opening Image: 一人で同じ片付けをしている。
- Catalyst: [ユーザー]がその儀式の意味に気づく。
- Midpoint: 儀式を二人で一度だけ共有する。
- All Is Lost: 共有した儀式が途切れる。
- Final Image: 同じ儀式を、違う距離でやり直す。

#### [ヒロインB] — 秘密の段階開示
- Opening Image: 表の役割だけが見えている。
- Catalyst: 役割と矛盾する小さな反応が出る。
- Midpoint: 秘密の一部を話し、関係が近づいたように見える。
- All Is Lost: 話したことで、別の境界線が露呈する。
- Final Image: 役割を保ったまま、隠し方だけが変わる。

#### [ヒロインC] — 境界線の再定義
- Opening Image: 近づけない距離を保っている。
- Catalyst: 近づく理由ではなく、止まる理由が共有される。
- Midpoint: 止まれるから近づける、という偽の勝利。
- All Is Lost: 境界線を破られた/破りかけた怖さが出る。
- Final Image: 同じ距離でも、確認できる関係になる。

### 重要なポイント
- LILIA で重要なのは **Catalyst / Midpoint / All Is Lost / Final Image** の4点。
- これらを意識するだけで物語の山と谷が作れる。
- 細かいビートは無視してよい。

### 使い時
- 関係が長期化してきて、起伏が必要だと感じた時。
- 中盤がだれてきた時 -> 今 Midpoint より前か後かを見る。

## 3. シノビガミの秘密構造 — 二層と争奪

TRPG「シノビガミ」のシナリオ設計から採れる構造。複数 PC の秘密と使命が絡む。LILIA は1人運用だが、構造の本質は「**公開される表**」と「**秘匿される裏**」の二層化、そして「**両者が欲する1つのもの**」の存在。

### 構成要素

| 要素 | 役割 |
|------|------|
| 使命（Mission） | 公開情報。「やるべきこと」「表向きの目的」 |
| 秘密（Secret） | 秘匿情報。「使命と矛盾する真実」「動機の真の姿」 |
| プライズ（Prize） | 関係者全員が欲する1つのもの。物でも人でも概念でもいい |
| 対立構造 | 使命と秘密が他のキャラと対立する形 |
| 調査判定 | 相手の秘密を暴くアクション。情報は徐々に開示される |

### 当てはめ例（構造説明のみ。literal として真似しないこと）

#### [ヒロインA] — 世話役
- 使命（公開）: 周囲の相談を受け続ける。
- 秘密（秘匿）: 自分が助けてほしいことを言えない。
- プライズ: 「頼っても壊れない関係」。
- 対立構造: 役割を守るほど本人の孤独が深くなる。
- 調査判定: [ユーザー]が答えを急がず、沈黙を共有する。

#### [ヒロインB] — 記録係
- 使命（公開）: 記録の整合性を守る。
- 秘密（秘匿）: 自分の記憶だけは信用できない。
- プライズ: 「記録に残らない本当の継続感」。
- 対立構造: 正確さと、関係の実感が衝突する。
- 調査判定: [ユーザー]が記録ではなく反応を覚えている。

#### [ヒロインC] — 境界線を守る人
- 使命（公開）: 近づきすぎない。
- 秘密（秘匿）: 近づきたいが、止まれなくなることが怖い。
- プライズ: 「近づいても止まれる安心」。
- 対立構造: 距離を守るほど、理解されたい気持ちが残る。
- 調査判定: [ユーザー]が確認し、待ち、断りを関係に残す。

### LILIA に取り入れる本質

1. **二層構造**: 表（使命）と裏（秘密）を別々に持つ。これは Background Truth で対応する。
2. **プライズ概念**: 関係者全員が欲する1つのもの。これが**関係の核**になる。
3. **調査による進行**: ユーザーの踏み込みが秘密を暴いていく構造。物語は「事件」ではなく「踏み込み」で進む。

### 使い時
- 関係の核が定まらないと感じた時 -> プライズを定義する。
- ユーザーが受動的になっていると感じた時 -> 「踏み込めば秘密が見える」構造になっているか確認する。

## 4. Ghost（ジョン・トルビーから1点だけ）

トルビーの22ステップは重い。1点だけ採る: **Ghost（過去の傷）**。

### 定義
- キャラの現在の弱点は、過去の傷（Ghost）の表れ。
- Ghost が今の選択を歪める。
- 物語のクライマックスで、Ghost と向き合うことで自己啓示が起きる。

### 当てはめ例（構造説明のみ。literal として真似しないこと）

- [ヒロインA]: かつて助けを求められなかった -> 今も「大丈夫」と言い続ける -> 頼ることが自己啓示になる。
- [ヒロインB]: 名前を変えた経験がある -> 今も署名欄で手が止まる -> 自分の呼ばれ方を選ぶことが自己啓示になる。
- [ヒロインC]: 境界線を失った経験がある -> 今も近づく前に逃げ道を探す -> 止まれる関係を信じることが自己啓示になる。

### LILIA への対応
- Background Truth に対応する概念。
- Ghost は本文で一気に説明しない。
- 現在の言動を**歪める**形で出し、Reveal Ladder で段階開示する。

### 使い時
- キャラが「動機が薄い」「行動の裏付けが弱い」と感じた時。
- Ghost を定義 or 強化する。

## 5. 起承転結 — 葛藤を作らずに物語を進める

日本古来の4幕構造。**葛藤を必須としない**。LILIA の静かな場面・slice of life 寄りに有効。

| 幕 | 役割 |
|---|------|
| 起 | 状況の提示 |
| 承 | 状況の発展 |
| 転 | 視点の転換、別の事実の挿入（葛藤ではなく） |
| 結 | 統合 / 余韻 |

### 当てはめ例（構造説明のみ。literal として真似しないこと）

#### [ヒロインA]と短い食事をする場面
- 起: [ヒロインA]が少し遅れて来る。
- 承: 料理や席の話をする。
- 転: 何気ない言葉で、いつも世話役でいる癖が見える。
- 結: 解決せず、次の第一声だけが変わる。

#### [ヒロインB]と資料を確認する場面
- 起: 同じ画面を見る。
- 承: 表の作業は進む。
- 転: 記録と本人の反応が少し食い違う。
- 結: 食い違いを説明せず、保留として残す。

#### [ヒロインC]と帰り道を共有する場面
- 起: 同じ方向へ歩く。
- 承: 距離を保ったまま雑談する。
- 転: 触れない選択が、拒絶ではなく信頼として見える。
- 結: 何も確定せず、次に会う余地だけ残る。

### 重要な点
- **「転」は葛藤ではなく、視点の転換**。
- 解決を求めない。余韻で終わってよい。
- LILIA は会話中心なので、日常場面では Save the Cat より起承転結が向く。

### 使い時
- 1場面（1セッションの1シーン）の組み立て。
- 「事件を起こさずに進めたい」場面。
- 物語が緊張しすぎている時、休符を入れたい時。

## 構造の使い分け早見表

| 状況 | 推奨構造 |
|------|---------|
| 関係全体の進行を診断したい | Story Circle |
| 長期的な山と谷を作りたい | Save the Cat |
| 表と裏の二層を設計したい | シノビガミ秘密構造 |
| プライズ（共有された欲望対象）を定義したい | シノビガミ秘密構造 |
| キャラの動機が弱いと感じる | Ghost |
| 静かな日常場面を組み立てたい | 起承転結 |
| 1場面の構成 | 起承転結 or Story Circle 縮約版 |

## 共通の原則

- 構造はテンプレではなく診断ツール。
- どの構造を使っても、結局は **「キャラが何を欲し、何を諦め、何に変わるか」** に帰結する。
- 構造があれば、「次の事件を作る」ではなく「次の段に進める」と考えられる。
- 構造を意識しすぎると場面が機械的になる。最終的な台詞や描写は構造から自由に。

## 出典
- Save the Cat: Blake Snyder, 2005 / Jessica Brody (Save the Cat! Writes a Novel), 2018
- Story Circle: Dan Harmon (Channel 101 / Wired記事 2011)
- The Anatomy of Story (22 Steps): John Truby, 2007（Ghost のみ採用）
- シノビガミ: 河嶋陶一朗 / 冒険企画局
- 起承転結: 漢詩起源、日本物語論で広く使用
